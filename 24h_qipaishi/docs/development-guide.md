# 开发指南

## 环境要求

- 微信开发者工具。
- Node.js/npm，用于安装小程序 npm 依赖。
- 微信小程序 AppID 和后端服务地址。
- 可用的后端 `/member/**` API。
- 如需调试蓝牙锁，需要微信插件 `wx43d5971c94455481` 及真实设备。

## 安装依赖

```bash
npm install
```

当前项目仅声明了 `@vant/weapp`。如果使用微信开发者工具，需要执行“工具 -> 构建 npm”，生成或刷新 `miniprogram_npm`。

## 关键配置

`app.js` 中当前写死：

```js
baseUrl: "https://wq.scyanzu.com/app-api"
tenantId: "150"
appName: "麻老板无人自助系统"
```

后续开发建议改为环境配置：

- 开发环境：dev API、测试租户。
- 预发环境：staging API、预发租户。
- 生产环境：prod API、真实租户。

## 本地调试流程

1. 使用微信开发者工具打开项目根目录。
2. 确认 `project.config.json` 中 AppID 与本地账号匹配。
3. 构建 npm，确保 Vant 组件可用。
4. 修改 `app.js` 的 `baseUrl` 和 `tenantId` 指向目标环境。
5. 在开发者工具中启用“不校验合法域名”或配置合法 request/upload 域名。
6. 使用微信登录流程获取手机号授权并缓存 token。
7. 选择门店，进入首页和下单流程。

## 测试重点

当前仓库没有自动化测试。手工回归建议覆盖：

- 首次进入未选门店时的门店选择。
- 微信手机号授权登录与 token 缓存。
- 房间时间轴展示、禁用时段、工作日价、通宵价。
- 优惠券、团购码、余额支付、微信支付四类支付路径。
- 支付成功、支付失败、支付回调延迟时的订单状态。
- 订单开始、续费、取消、换房、分享订单。
- 大门、房间门、远程锁、蓝牙锁兜底。
- 保洁接单、开始、开门、完成、投诉、结算。
- 管理员配置门店、房间、优惠券、充值规则。
- 统计图表在空数据、大数据、多门店场景下的展示。

## 后续工程化建议

- 增加 ESLint/Prettier 或项目统一格式。
- 抽取 API 契约和领域 service，减少页面直接写业务规则。
- 给价格计算、时间计算、订单状态流转增加单元测试。
- 引入 mock API 或小程序自动化测试用于关键预约链路。
- 移除硬编码租户和生产接口，避免误操作真实环境。
