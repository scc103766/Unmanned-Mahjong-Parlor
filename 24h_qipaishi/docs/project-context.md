# 项目上下文：24H 无人棋牌室管理系统

## 背景

当前仓库 `24h_qipaishi` 是一个微信原生小程序参考项目，面向 24 小时无人自助棋牌室/共享茶室场景。现有代码覆盖顾客预约下单、订单支付、订单详情、一键开门、房间与门店管理、保洁任务、会员余额、优惠券、团购码核销、经营统计等能力。

参考项目本身主要是小程序端；服务端、数据库、硬件设备接入与管理后台不在当前仓库内。后续开发需要在保留参考项目业务经验的基础上，重新定义完整产品边界，包括小程序端、服务端 API、管理端、硬件接入、支付结算与运营数据。

## 技术现状

- 小程序框架：微信原生小程序，JS/WXML/WXSS。
- UI 组件：`@vant/weapp`。
- 图表：`ec-canvas` + ECharts。
- 时间处理：本地 `lib/moment.js`。
- 网络层：`utils/http.js` 封装 `wx.request`，统一添加 `tenant-id` 和 `Authorization: Bearer <token>`。
- 当前后端地址：`app.globalData.baseUrl = https://wq.scyanzu.com/app-api`。
- 当前租户标识：`app.globalData.tenantId = "150"`。
- 硬件能力：通过 REST API 触发远程开门/开电；通过微信插件 `wx43d5971c94455481` 处理蓝牙门锁本地开锁兜底。

## 核心业务域

- 顾客域：登录、选店、查看房间、预约、支付、订单详情、续费、取消、开门、查看余额和优惠券。
- 门店域：门店资料、门店图片、地图坐标、客服信息、公告、Wi-Fi、营业参数。
- 房间域：房间类型、价格、工作日价、通宵价、最小时长、提前预约、不可用时段、清洁时间、标签、房间图片、门锁配置。
- 订单域：预下单、锁定微信支付订单、保存订单、开始订单、续费、换房、取消、退款、订单分享、团购码核销。
- 设备域：大门、房间门、电源/门禁、蓝牙锁、远程网关、云喇叭语音。
- 保洁域：保洁账号、任务列表、接单、开始、开门、完成、投诉、结算、统计。
- 运营域：管理员、会员、优惠券、充值规则、提现、订单统计、收益统计、房间使用率、会员统计。

## 后续开发原则

- 以“24 小时无人运营闭环”为主线，不只复刻小程序页面。
- 硬件操作必须有权限校验、订单状态校验、审计日志和失败兜底。
- 支付、开门、续费、退款、清洁状态变化必须做幂等处理。
- 多门店、多租户能力从数据模型层设计，不依赖前端硬编码。
- 管理端和小程序端共享业务规则，但分离权限边界。
- 文档、PRD、架构和故事均放入 BMAD 输出目录，便于后续按 BMAD 继续拆解。
