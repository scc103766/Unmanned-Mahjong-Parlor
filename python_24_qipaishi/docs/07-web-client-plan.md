# 网页端开发计划

## 目标调整

项目当前目标调整为先开发网页端系统，完成无人棋牌室的业务闭环和后台运营能力；App 不再作为近期主线，后续在业务稳定后再深化移植为手机端和 PC 端 App。

第一阶段产品形态：

- 顾客 H5/移动网页：选店、选房、预约、支付、订单详情、开门、续费、取消。
- 商家 Web 后台：门店、房间、价格、设备、订单、会员、营销、保洁、财务和统计。
- 保洁移动网页：任务列表、接单、开始、开门、上传凭证、完成和结算记录。
- 平台 Web 后台：租户、权限、设备供应商、异常、审计和人工补偿。
- Python 后端：统一承接租户、权限、订单、支付、设备、保洁、资金、统计和审计。

参考小程序继续作为业务流程和字段语义来源；Flutter App 目录保留为后续迁移储备，不参与近期主线验收。

## 推荐前端技术栈

建议采用 Vue 3 + Vite + TypeScript 作为网页端主栈：

| 层次 | 建议 |
| --- | --- |
| 框架 | Vue 3 |
| 构建 | Vite |
| 语言 | TypeScript |
| 路由 | Vue Router |
| 状态 | Pinia |
| 请求 | Axios 或基于 Fetch 的轻封装 |
| UI | Element Plus 用于后台，Vant 或 NutUI 用于移动网页 |
| 表格图表 | Element Plus Table、ECharts |
| 测试 | Vitest、Playwright |
| 代码质量 | ESLint、Prettier |

选择理由：

- 网页端开发和部署比 App 更快，适合先验证业务闭环。
- 移动网页可覆盖顾客和保洁端的现场流程。
- Web 后台更适合商家和平台运营的高密度表格、筛选和批量操作。
- 后续迁移 App 时，稳定后的 API、业务状态和页面流程可复用为产品规格。

## 页面范围

### 顾客移动网页

- 登录/手机号绑定。
- 城市/门店选择。
- 门店详情、公告、客服、地址和导航。
- 房间列表、图片、标签、价格、可预约时间轴。
- 下单、价格明细、优惠券、团购码、余额和支付入口。
- 订单列表、订单详情、开店门、开房门、开电。
- 续费、取消、换房、帮助中心。

### 保洁移动网页

- 任务列表和状态筛选。
- 任务详情。
- 接单、取消接单、开始任务。
- 任务授权开门。
- 上传清洁凭证。
- 完成任务、查看驳回原因和结算记录。

### 商家 Web 后台

- 工作台：今日订单、房态、待支付超时、待保洁、设备异常、退款待处理。
- 门店管理：门店资料、公告、Wi-Fi、客服、地图坐标、图片。
- 房间管理：房间资料、标签、图片、价格、工作日价、通宵价、禁用时段。
- 设备管理：店门、房间门、智能锁、电源、云喇叭、设备命令记录。
- 订单中心：列表、详情、取消、退款、改时、续费、换房、人工完成。
- 会员营销：会员、余额、充值规则、优惠券、团购码核销。
- 保洁管理：任务派发、审核、投诉、驳回、结算。
- 财务：支付、退款、余额流水、提现、对账导出。
- 统计：收入、订单、退款、充值、房间使用率、保洁效率、设备成功率。

### 平台 Web 后台

- 租户管理。
- 用户、角色、门店作用域。
- 应用绑定和支付配置。
- 设备供应商配置。
- 异常列表和人工补偿。
- 审计查询。

## API 策略

网页端直接使用 `/api/v1/**` 领域 API：

```text
Customer H5 / Cleaner H5 / Merchant Web / Platform Web
  -> FastAPI /api/v1/**
  -> PostgreSQL / Redis / Worker
```

兼容层 `/compat/member/**` 仅作为参考小程序迁移辅助，不作为网页端主接口。

## 开发阶段

| 阶段 | 目标 | 交付 |
| --- | --- | --- |
| W0 Web 底座 | 创建 Vue/Vite/TS 工程、路由、请求、鉴权、布局 | 可启动 Web 空壳、API client、登录态、权限守卫 |
| W1 顾客闭环 | 顾客移动网页完成选店、选房、下单、订单详情、开门 mock | 顾客 H5 MVP |
| W2 商家基础后台 | 门店、房间、价格、订单列表和工作台 | 商家 Web MVP |
| W3 设备与保洁 | 设备命令、保洁任务、保洁移动网页 | 保洁和设备闭环 |
| W4 支付营销财务 | 支付、余额、优惠券、团购码、退款、提现 | 资金营销闭环 |
| W5 平台运营统计 | 租户、角色、异常、审计、统计 | 平台后台与统计 |
| W6 联调试运营 | 压测、安全、权限、告警、试运营修复 | 上线清单和试运营版本 |

## 后续 App 迁移策略

App 迁移不废弃当前工作，而是复用网页端沉淀：

- 复用稳定的 `/api/v1/**`。
- 复用页面流程、权限边界、错误码、状态机和字段口径。
- 顾客和保洁端优先迁移为手机 App。
- 商家和平台后台可长期保留 Web，必要时再做 PC App。
- 当前 Flutter 空壳保留在 `clients/qipaishi_app`，但从近期验收中移出。

## 当前验收标准

近期验收以网页端为准：

- Web 工程可启动，商家后台首版已完成 Vue 3 + Vite + TypeScript 正式工程化。
- 顾客 H5 能跑通预约订单 mock 闭环。
- 商家后台能查看运营总览、订单、会员、提现、异常和审计，并可对接后端处理改时、补偿和提现。
- 保洁移动网页能处理任务。
- 平台后台能查看租户、异常和审计。
- 所有高风险操作仍由后端权限和审计裁决。
