# Hermes Agent 能力使用规划

## 定位

`hermes-agent` 在本项目中优先作为研发自动化、运维巡检、运营分析和人工辅助工具使用，不作为订单、支付、库存、设备控制等核心业务链路的最终裁决者。

当前主线仍是 Python 后端 + 网页端：

- FastAPI 负责业务规则、状态机、权限、幂等、审计和外部供应商适配。
- Web 端负责顾客 H5、保洁移动网页、商家后台和平台后台交互。
- Hermes Agent 可在开发期提高产出效率，在试运营期提供只读分析和审批前草案，在业务稳定后再作为 Web 智能助手能力的一部分。
- 项目 Python 运行环境继续使用 `anti-spoofing_scc_175`；Hermes 不应改变项目业务环境。

## 建议嵌入点

后续如果把 Hermes 能力接入业务系统，建议先以独立模块方式接入：

```text
app/
  modules/
    assistant/
      conversations.py
      tool_calls.py
      approvals.py
      policies.py
    ops_insights/
      order_insights.py
      device_insights.py
      cleaning_insights.py
      finance_insights.py
  providers/
    hermes/
      client.py
      prompts.py
      schemas.py
web/
  src/modules/platform/assistant/
  src/modules/merchant/assistant/
  src/modules/support/assistant/
```

第一版业务集成只开放只读工具：

- 查询订单、支付单、退款单、设备命令、保洁任务和审计日志。
- 汇总异常原因、生成处理建议、创建工单草案。
- 生成日报、周报、门店经营摘要和设备供应商稳定性摘要。

第二版再开放审批型写操作：

- 草拟退款、补偿、改时、清洁催办、设备重试。
- 由人工在 Web 后台确认后执行。
- 执行时必须走后端原有服务、幂等键、权限校验和审计日志。

## 不建议交给 Agent 自主裁决的部分

以下能力必须由确定性后端服务裁决，Hermes 最多只能解释、汇总或生成草案：

- 房间库存锁和可预约性最终判断。
- 订单状态机流转。
- 微信支付回调确认、查单确认和资金流水落账。
- 退款、提现、结算的最终审批和执行。
- 开店门、开房门、开锁、开电等设备命令的直接执行。
- 租户隔离、RBAC、门店作用域和数据权限。
- 价格计算、优惠券核销、团购码核销和余额扣减。
- 保洁结算金额、财务对账金额和商家分账金额。

## 权限与审计原则

- 默认只读：Agent 首先只能读经过授权的聚合数据和日志。
- 最小作用域：每次工具调用必须带 `tenant_id`、`actor_id`、`role`、`store_scope` 和 `request_id`。
- 人工确认：资金、设备、订单、权限相关写操作必须由 Web 后台人工确认。
- 全量审计：Agent 的输入、输出、工具调用、审批人、执行结果都要写入审计。
- 失败关闭：Agent 调用失败时不能绕过后端业务规则，不能自动降级为无审计操作。
- 敏感数据脱敏：手机号、支付标识、设备密钥、供应商 token 和定位信息按最小必要展示。

## 分阶段落地

| 阶段 | 目标 | 交付 |
| --- | --- | --- |
| H0 研发期 | 用 Hermes 辅助开发、测试、文档和代码审查 | 代码、迁移、测试、文档的开发辅助流程 |
| H1 后端核心期 | 辅助 M1-M6 模块实现和状态机测试 | 租户、订单、支付、设备、保洁测试用例和排障记录 |
| H2 运营只读期 | 建立只读运营分析和异常摘要 | 日报、异常摘要、设备失败统计、保洁积压统计 |
| H3 Web 助手期 | 在平台/商家后台加入助手入口 | 只读问答、工单草案、处理建议 |
| H4 审批动作期 | 支持审批后执行低风险动作 | 清洁催办、设备重试草案、退款/补偿草案 |
| H5 App 迁移期 | 将成熟助手能力迁移到移动端 | 顾客/保洁/商家移动助手能力 |

## 近期可执行清单

1. 开发期使用 Hermes 辅助生成 FastAPI 模块、迁移、测试和 Web 页面，但所有改动仍以本项目测试命令验收。
2. 在 M5 设备模块完成后，设计只读的设备命令异常摘要接口。
3. 在 M6 保洁模块完成后，设计保洁任务积压和 SLA 摘要接口。
4. 在 M7 统计模块完成后，提供经营日报数据源。
5. 在 M8 平台后台加入“异常助手”页面草案，先只展示分析结果和人工处理建议。
6. 在试运营前补齐 Agent 调用审计表、审批流和敏感数据脱敏策略。
